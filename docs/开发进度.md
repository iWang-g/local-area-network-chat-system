## 2. 服务端 `vs-server`（已实现）

### 2.1 网络与协议

- **WinSock 2** TCP 监听、`accept`、每连接独立线程处理（`tcp_server.cpp`）。
- **成帧**：4 字节大端长度 + UTF-8 JSON；`FrameAssembler` 处理半包/粘包；单帧上限 256 KiB（`wire.*`）。
- **握手**：校验 `type=hello`、`magic=LNCS`、`version=1`；成功回复 `hello_ok`，失败 `error`（code `1001`）。
- **心跳**：`heartbeat` / `heartbeat_ack`；未完成握手时返回 `error`（code `2010`）。
- **认证**：`auth_login`、`auth_register`（必填 `email_code`）、`req_email_code`（`purpose=register`）；成功回复 `auth_ok` / `email_code_ok`；错误码含 `2006`/`2007`/`2008`（见协议文档）。  
- **mail-helper**：可选环境变量 **`LANCS_MAIL_HELPER_URL`**（完整路径，如 `http://127.0.0.1:8788/v1/send-register-code`）、**`LANCS_MAIL_HELPER_SECRET`**（须与辅助服务一致）；`mail_helper_http`（WinHTTP）POST JSON；失败回滚验证码并返回 `2008`。
- **JSON 辅助**：`message_parse` 中 `type` 解析、字符串字段提取、`error`/`auth_ok` 拼装及转义。

### 2.2 数据库与安全

- **SQLite**：运行时加载 **`sqlite3.dll`**（`sqlite_dynamic`），官方 **x64** DLL 置于 `third_party/sqlite/`，VS **x64** 生成后事件复制到输出目录；可选脚本 `scripts/fetch-sqlite-dll.ps1` 重新下载。
- **数据文件**：`exe` 同目录下 `data/chat.db`；`PRAGMA foreign_keys=ON`；**`users`**（表 4-1）、**`email_codes`**（表 4-2）（`AppDatabase`）。
- **口令**：随机盐 + 迭代 SHA256（`password_hash` + PicoSHA2）；库中仅存哈希与盐，无明文。
- **会话 token**：登录/注册成功后生成 hex token，内存中 `token → user_id` 映射（后续业务报文校验可扩展）。

### 2.3 工程与其它

- **日志**：控制台 + 按日落盘 `logs/YYYY-MM-DD.log`；Windows 下控制台 UTF-8 与日志 UTF-8 BOM，减轻中文乱码（`logger.cpp`）。
- **入口**：`main.cpp` 端口参数、数据库初始化与关闭；启动时打印 **`LANCS_MAIL_HELPER_URL` 是否已设置**（便于确认 VS / 终端环境变量是否生效）。

## 3. 客户端 `qt-client`（已实现）

- **`LanTcpClient`**：`QTcpSocket`、长度前缀编解码、JSON 解析与信号 `jsonReceived` / `protocolError`。
- **登录页**：仿 QQ 风格卡片布局（`LoginWidget` + `AppStyle`）；**服务器地址 / 端口**（`QSettings`：`server/host`、`server/port`）。
- **主流程**：登录或注册 → 连接 → `hello` → `auth_login` / `auth_register` → `auth_ok` 后进入主界面；错误信息展示服务端 `message`。
- **注册**：独立注册页；**获取验证码** → `req_email_code`（`email` 为收件地址）；填写 **邮箱验证码** 后注册；未配置 `LANCS_MAIL_HELPER_URL` 时验证码见**服务端日志**；已配置时由 **mail-helper** 发信（`SMTP_URL` 为空时辅助服务 dry-run 仅打日志）。
- **主窗口**：`QStackedWidget` 切换登录页与聊天工作区；菜单「TCP 联调」、`NetworkDebugDialog` 便于抓包调试；聊天区为后续阶段占位（`ChatMainWidget` 等）。

## 4. 邮件辅助服务 `mail-helper/`（已实现）

- **技术栈**：Node.js（建议 ≥18）、Express、nodemailer、dotenv；根目录 **`mail-helper/`**，`npm start` 启动（PowerShell 若拦截 `npm.ps1` 可设执行策略或改用 cmd / `npm.cmd`）。
- **接口**：`GET /health`；`POST /v1/send-register-code`，JSON 体 `email`、`code`、`purpose`（默认 `register`）；可选 **`Authorization: Bearer`**，与 **`MAIL_HELPER_SECRET`** / 服务端 **`LANCS_MAIL_HELPER_SECRET`** 一致。
- **配置**：复制 **`.env.example`** 为 **`.env`**。核心项：**`SMTP_URL`**（如 QQ：`smtps://邮箱:授权码@smtp.qq.com:465`，授权码中特殊字符需 URL 编码）、**`MAIL_FROM`**（发件人，需与 SMTP 账号一致）、**`PORT`**（默认 8787；若改为 8788 等，须同步修改 `LANCS_MAIL_HELPER_URL`）。
- **行为**：`SMTP_URL` 未设置时返回 200 且 **`dryRun: true`**，控制台打印验证码，便于不发信联调；已设置则真实 SMTP 发信，控制台输出 **`recv POST`** / **`sent register code`** 或错误栈。

## 6. 已知限制与近期可做事项

- **运维与安全**：未接 mail-helper 时服务端日志仍含**明文验证码**；答辩/生产可关闭或脱敏。`MAIL_HELPER_SECRET` 建议在局域网演示时开启。
- **环境变量**：PowerShell **`$env:...` 仅当前窗口**；VS **调试**须在 **项目属性 → 调试 → 环境** 中配置，格式示例：`LANCS_MAIL_HELPER_URL=http://127.0.0.1:8788/v1/send-register-code`（**等号两侧勿加空格**，URL 写全路径）；**合并环境**建议为「是」。
- **token**：已下发，尚未在除登录/注册外的报文中强制校验。
- **服务端可移植性**：当前网络层为 Windows 专用；Linux 需替换为可移植实现（路线中已注明）。**mail-helper** 可独立部署于 Linux，只要把 `LANCS_MAIL_HELPER_URL` 指向可达地址。
- **可选优化**：主流程与联调对话框共用连接时的状态机边界、更细的协议字段（如 `seq`）等。
