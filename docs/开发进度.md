## 2. 服务端 `vs-server`（已实现）

### 2.1 网络与协议

- **WinSock 2** TCP 监听、`accept`、每连接独立线程处理（`tcp_server.cpp`）。
- **成帧**：4 字节大端长度 + UTF-8 JSON；`FrameAssembler` 处理半包/粘包；单帧上限 256 KiB（`wire.*`）。
- **握手**：校验 `type=hello`、`magic=LNCS`、`version=1`；成功回复 `hello_ok`，失败 `error`（code `1001`）。
- **心跳**：`heartbeat` / `heartbeat_ack`；未完成握手时返回 `error`（code `2010`）。
- **认证**：`auth_login`、`auth_register`；成功回复 `auth_ok`（`user_id`、`token`、`email`）；错误码 `2001`～`2005` 等（见协议文档）。
- **JSON 辅助**：`message_parse` 中 `type` 解析、字符串字段提取、`error`/`auth_ok` 拼装及转义。

### 2.2 数据库与安全

- **SQLite**：运行时加载 **`sqlite3.dll`**（`sqlite_dynamic`），官方 **x64** DLL 置于 `third_party/sqlite/`，VS **x64** 生成后事件复制到输出目录；可选脚本 `scripts/fetch-sqlite-dll.ps1` 重新下载。
- **数据文件**：`exe` 同目录下 `data/chat.db`；`PRAGMA foreign_keys=ON`；**`users` 表**对齐论文表 4-1（`AppDatabase`）。
- **口令**：随机盐 + 迭代 SHA256（`password_hash` + PicoSHA2）；库中仅存哈希与盐，无明文。
- **会话 token**：登录/注册成功后生成 hex token，内存中 `token → user_id` 映射（后续业务报文校验可扩展）。

### 2.3 工程与其它

- **日志**：控制台 + 按日落盘 `logs/YYYY-MM-DD.log`；Windows 下控制台 UTF-8 与日志 UTF-8 BOM，减轻中文乱码（`logger.cpp`）。
- **入口**：`main.cpp` 端口参数、数据库初始化与关闭。

## 3. 客户端 `qt-client`（已实现）

- **`LanTcpClient`**：`QTcpSocket`、长度前缀编解码、JSON 解析与信号 `jsonReceived` / `protocolError`。
- **登录页**：仿 QQ 风格卡片布局（`LoginWidget` + `AppStyle`）；**服务器地址 / 端口**（`QSettings`：`server/host`、`server/port`）。
- **主流程**：登录或注册 → 连接 → `hello` → `auth_login` / `auth_register` → `auth_ok` 后进入主界面；错误信息展示服务端 `message`。
- **注册**：复用邮箱/密码输入，昵称通过对话框采集；**未接邮箱验证码**（与阶段 3 一致）。
- **主窗口**：`QStackedWidget` 切换登录页与聊天工作区；菜单「TCP 联调」、`NetworkDebugDialog` 便于抓包调试；聊天区为后续阶段占位（`ChatMainWidget` 等）。

## 5. 已知限制与近期可做事项

- **阶段 3**：注册未校验邮箱验证码；无 `email_codes` 表业务、无邮件辅助服务。

- **可选优化**：主流程与联调对话框共用连接时的状态机边界、更细的协议字段（如 `seq`）等。
