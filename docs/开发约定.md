# 开发约定

## 1. 工程与工具链

| 组件 | 打开方式 | 注意 |
|------|-----------|------|
| `qt-client` | Qt Creator → 打开 `qt-client/CMakeLists.txt` | Qt 套件须与编译器一致（如 MSVC 对应 msvc2019_64），勿混用 MinGW Qt + MSVC |
| `vs-server` | Visual Studio → `vs-server/vs-server.sln` | C++17、`/utf-8` 已在工程层配置 |

## 2. 技术选型（固定）

- 客户端 UI：**Qt Widgets**，**不使用 QML / Qt Quick**（除非课题范围变更）。  
- 客户端网络：`QTcpSocket`（或同类异步 API），**主线程不阻塞**长耗时 IO。  
- 服务端：标准 C++ + Socket + 多线程或事件模型（实现阶段定稿并写在代码注释或本文件）。

## 3. 编码与风格

- 源文件：**UTF-8**。  
- 字符串：Qt 侧优先 `QStringLiteral` / `u8` 等明确编码的写法；服务端日志避免乱码。  
- 命名：与现有目录一致（`src/ui`、`src/style`、`include/vsserver`）；新增模块用**完整单词**，避免过度缩写。

## 4. 线程与异步

- UI 线程：**不做**大量计算、**不做**阻塞式网络/磁盘。  
- 服务端：共享数据结构（在线用户表、连接映射）必须有**互斥或专用线程**策略，避免数据竞争。

## 5. 日志与调试

- **qt-client**：`Logger::init()` / `shutdown()`（见 `src/utils/logger.*`），安装 `qInstallMessageHandler`，**stderr 同步输出**并**按日追加**到 `qt-client/logs/yyyy-MM-dd.log`（CMake 定义 `PROJECT_ROOT_DIR` 指向工程根；无定义时退化为可执行文件目录）。日常请用 `qDebug` / `qInfo` / `qWarning` / `qCritical`。  
- **vs-server**：`vsserver::Logger::init()` / `shutdown()`（见 `include/vsserver/logger.hpp`、`src/utils/logger.cpp`），宏 **`VSLOG_DEBUG` / `VSLOG_INFO` / `VSLOG_WARN` / `VSLOG_ERROR`**，输出到 **stderr** 与 **`{当前工作目录}/logs/yyyy-MM-dd.log`**（`init` 可传入自定义根目录）。`tcp_server` 等已改用上述宏。  
- 仓库根 `.gitignore` 已忽略 `logs/`，避免把运行日志提交进 Git。  
- 关键路径：连接建立/断开、登录失败原因（脱敏）、协议解析错误。  
- 生产/演示环境：**勿**在日志中输出完整密码、完整 token、验证码明文（调试级别可开关）。

## 6. 文档同步

- 协议类型、错误码、表结构变更：先改**实现与 `docs/`**，论文大改留到阶段稳定后统一合并。  
- 若行为与论文流程图不一致，在 `docs/` 增加**“与论文差异”**一小节说明原因。
