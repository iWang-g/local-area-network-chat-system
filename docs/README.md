# 项目文档（开发导向）

本目录存放**实现阶段**使用的说明与路线，与《毕业设计》正文分工如下：

| 材料 | 定位 |
|------|------|
| **`毕业设计/`** | 需求、总体设计、数据库与流程的**正式表述**（论文定稿） |
| **`docs/`** | 仓库落地时的**迭代顺序、目录约定、协议草案、联调要点**，可随开发更新 |

**建议阅读顺序**

1. [架构与代码布局](./架构与代码布局.md) — 谁做什么、工程怎么拆  
2. [协议与接口草案](./协议与接口草案.md) — 客户端 / 主服务端 / 邮件辅助服务怎么说话  
3. [数据库与持久化要点](./数据库与持久化要点.md) — 表用途与实现时注意点（细节以论文 4.3 为准）  
4. [开发约定](./开发约定.md) — 工具链、编码、线程与日志  
5. [开发路线](./开发路线.md) — 分阶段里程碑与验收标准（**含仿 QQ 风格的 UI 设计开发**，与各阶段穿插）  
6. [UI参考](./UI参考.md) — `ui参考图/` 目录说明与**参考图索引表**（新增 PNG 时请同步登记）  

论文中的用例图、流程图、E-R 图仍以 `毕业设计/` 与 `毕业设计/figures/` 为准；若实现与论文有微调，优先在 `docs/` 记录**差异与原因**，必要时再回溯改论文。
