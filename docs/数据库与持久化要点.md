# 数据库与持久化要点

## 1. 权威来源

**表名、字段、约束、E-R 关系**以《毕业设计》**第 4.3 节**及其中表格为准。本文只强调实现时容易忽略的点。

## 2. 引擎与文件

- **SQLite** 单文件库，由**主服务端进程**独占打开（WAL 可按需开启）。  
- 启动时执行：`PRAGMA foreign_keys = ON;`（若使用外键）。  
- 备份：直接拷贝 `.db` 文件需**停写**或使用 SQLite 备份 API，避免半写损坏。

## 3. 核心表职责（速查）

| 表（论文中的逻辑名） | 用途 |
|----------------------|------|
| 用户表 | 账号、邮箱、密码哈希、盐、昵称、头像路径等 |
| 验证码相关 | 邮箱验证码、过期时间、尝试次数 |
| 好友申请 | 申请状态、双方用户 ID |
| 好友关系 | 双向关系或两条有向边（以实现选型为准） |
| 消息 | 点对点消息内容、时间、送达/已读标记（若实现） |
| 文件传输记录 | 分片会话、进度、校验信息（支持断点/排查） |

## 4. 安全与一致性

- **禁止**明文存密码；使用成熟哈希算法 + **每用户独立盐**。  
- 验证码：**短有效期**、**单邮箱发送间隔**、**单日次数上限**。  
- 删除好友、拒绝申请等操作使用**事务**，避免关系表与申请表不一致。

## 5. 与客户端的关系

客户端**不直接访问** SQLite；所有查询与写入经 **TCP 协议** 由服务端完成。若客户端做本地缓存（可选），必须以**服务端数据为准**做同步与纠错。
