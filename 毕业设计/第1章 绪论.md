# 第1章 绪论

## 1.1 研究背景

在当前信息化社会中，计算机网络已成为连接世界的桥梁，而局域网（Local Area Network, LAN）作为特定地理范围内的网络基础设施，在校园实验室、小型企业、政府机构等场景中扮演着至关重要的角色。这些场景对内部信息交互的**私密性、实时性与安全性**有着极高的要求。传统的沟通方式，如面对面交流、电话、电子邮件等，在信息传递效率和即时性方面存在局限。随着即时通讯技术的普及，人们习惯于通过即时通讯软件进行沟通，但将公网即时通讯工具直接应用于局域网内部通信，则面临诸多挑战 [1]。

首先，主流的公网即时通讯软件功能庞杂，存在大量与局域网内部通信无关的冗余功能，不仅增加了软件的复杂性，也可能占用不必要的系统资源。其次，公网通讯工具的数据传输和存储依赖于外部服务器，这在追求高度私密性的局域网环境中带来了潜在的**隐私泄露风险**。对于校园实验室和小型企业而言，内部数据往往涉及科研成果、商业机密等敏感信息，一旦泄露将造成不可估量的损失。此外，部分简易的局域网聊天工具虽然避免了公网依赖，但普遍存在功能单一（仅支持基础文字消息）、跨平台适配性差（仅支持单一操作系统）以及缺乏有效的账号身份核验机制等问题，难以满足日益增长的复杂通信需求 [2]。

鉴于上述背景，开发一款**基于 C/S（Client/Server）模式的局域网聊天系统**显得尤为必要。该系统应充分利用局域网的低延迟、高带宽优势：**客户端**采用 **C++** 与 **Qt 6** 图形框架，实现跨平台界面与网络交互；**服务端**采用标准 **C++**，基于 **TCP/IP Socket** 与**多线程**完成连接管理、鉴权、消息路由与数据访问，并支持在 **Windows、Linux** 等环境下部署；**SQLite** 承担用户、好友关系与聊天记录等数据的轻量级持久化；注册登录可结合**邮箱验证码**机制，邮件投递依托 **SMTP**，并可通过**独立辅助服务**完成发信以降低主服务端复杂度。由此构建专门服务于局域网内部的、集安全性、高效性与便捷性于一体的即时通讯平台，替代传统低效的沟通方式，提升内部信息交互的质量与效率，并更好地保障通信的私密性与可控性，从而满足校园实验室、小型企业等场景的特定需求。

## 1.2 研究意义

本课题基于 C/S 模式的局域网聊天系统研究与实现，兼具实际应用价值与技术参考意义。系统聚焦校园实验室、小型企业的局域网私密通信需求：**客户端**依托 **Qt 6** 实现图形界面与交互体验，**服务端**以 **C++** 与 **Socket** 通信为核心并在多系统环境下可部署，从而打破单一平台限制；通过邮箱验证码注册登录与业务数据在局域网内传输、落地，提升身份可信度与私密性；服务端结合多线程等手段缓解多用户并发场景下的阻塞与资源争用，增强通信稳定性；在此基础上实现即时消息、大文件传输、聊天记录追溯等核心功能，优化局域网内部沟通效率，为局域网私密通信提供实用可行的解决方案，也为同类 C/S 架构跨平台通信系统开发提供技术参考。

## 1.3 国内外研究现状

### 1.3.1 国内研究现状

在国内，基于 C/S 模式的局域网聊天系统研究已取得显著进展，研究者们围绕架构优化、网络通信稳定性及跨平台适配等方面展开了大量实践。

1.  **C/S 架构与 Socket 通信的深度优化**：在网络通信基础构建方面，李向江（2024）在《基于 Socket 接口的局域网聊天系统设计》一文中，指出利用 TCP/IP 协议栈能够实现高可靠性的点对点通信，为局域网内的即时消息传输提供了坚实的协议基础 [1]。在系统安全与认证方面，张玉（2022）在《基于 Socket 的网络聊天系统的设计与实现》一文中，通过引入身份认证机制提升了 C/S 模式下的数据传输安全性，为局域网账号核验提供了理论依据 [9]。针对特定场景的应用，沈薇莲（2013）在《基于 C/S 模式的校内局域网聊天系统的开发设计》一文中，结合校内局域网环境验证了 C/S 模式在处理内部私密信息交互时的结构优势 [3]。

2.  **Qt 框架与跨平台开发实践**：在跨平台技术应用方面，唐天波（2018）在《基于 Linux 的局域网聊天软件设计与实现》一文中，详细阐述了利用 Qt 开发局域网聊天软件的过程，重点解决了图形界面与底层 Socket 通信的解耦问题 [2]。针对国产操作系统的适配，王苏敏（2012）在《Linux 系统下基于 QT 的局域网聊天系统》一文中，通过优化网络数据包解析效率，提升了 Qt 应用在 Linux 环境下的运行流畅度 [5]。在多系统兼容性探索上，吴捷（2014）在《Linux 下基于 Qt4 的跨平台聊天系统设计》一文中，证明了 Qt 框架在处理不同操作系统网络协议差异时的灵活性与高效性 [18]。

3.  **高并发处理与多线程应用**：在提升系统并发能力方面，王林（2020）在《基于 C/S 模式的高并发局域网聊天系统设计》一文中，提出利用线程池技术降低服务器在处理大规模并发请求时的 CPU 占用率，显著优化了系统性能 [11]。在异步通信机制研究上，韩国福（2025）在《基于 Qt 高并发的 TCP 通信机制设计》一文中，通过异步非阻塞 I/O 模型解决了传统同步套接字带来的通信阻塞问题，为多用户并发连接提供了技术保障 [16]。

4.  **功能集成与安全性增强**：在功能拓展方面，毕娜（2016）在《局域网聊天室系统的设计与实现》一文中，引入了文件共享与实时状态监控功能，丰富了局域网聊天系统的应用场景 [17]。在数据隐私保护上，温宇飞（2021）在《跨平台的聊天系统设计与实现》一文中，探讨了数据加密传输对保护局域网内敏感信息的重要性 [12]。

### 1.3.2 国外研究现状

国外在计算机网络技术和即时通讯协议的研究上起步较早，其成果为局域网聊天系统的开发提供了成熟的技术支撑和架构参考。

1.  **即时通讯系统的搜索与索引技术**：在提升用户交互体验方面，R. R. P. K.（2019）在《Design and Implementation of Search-Box in Instant Messaging Application》一文中，提出了一种高效的搜索框设计方案，实现了对历史聊天记录的快速检索，极大提升了信息获取效率 [14]。

2.  **架构轻量化与 Java 技术的应用**：在跨平台部署逻辑上，Lu C.（2017）在《Design and Implementation of the Instant Messaging Tool Based on JAVA》一文中，强调了 C/S 架构在不同运行环境下代码复用性的重要价值，其功能模块划分思路具有广泛的借鉴意义 [15]。

3.  **安全通信模型的研究**：在安全协议设计方面，L. Zhang（2022）在《Research on the Secure Communication Model of Instant Messaging》一文中，对主流即时通讯应用的安全模型进行了深度分析，涵盖了消息加密流程及身份验证逻辑，为局域网安全机制设计提供了逻辑框架 [12]。

4.  **实时性与网络效率优化**：在网络性能评估上，相关研究如《Real-Time Chat Application with Web Socket for Network Efficiency》（2025）探讨了利用 Web Socket 技术提升网络效率的可能性，为 C/S 模式下的网络指标优化提供了新思路 [14]。

## 1.4 主要研究内容

本课题“基于 C/S 模式的局域网聊天系统设计与实现”的主要研究内容涵盖了从需求分析到系统测试的完整开发生命周期，具体包括以下几个方面：

1.  **需求分析**：明确项目的开发目标和核心功能需求，包括邮箱注册登录、好友添加与管理、点对点消息传输、文件传输、聊天记录存储与查询、好友状态实时显示等，以满足局域网内的私密通信需求。

2.  **系统设计**：
    *   **架构设计**：设计 C/S 模式的整体系统架构，明确客户端与服务器端的功能模块划分及交互逻辑。
    *   **数据库设计**：设计基于 SQLite 的数据库结构，包括用户表、好友关系表、消息表、文件传输记录表等，实现数据的持久化存储与管理。
    *   **网络通信协议设计**：制定客户端与服务器之间的通信协议，确保数据传输的规范性和可靠性。
    *   **界面设计**：设计简洁直观、操作便捷的客户端界面，包括登录注册、好友列表、聊天窗口、文件传输、聊天记录查询等。

3.  **系统实现**：
    *   **服务端开发**：使用 C++ 搭建服务器端，实现客户端连接监听、数据路由与转发、用户鉴权、数据读写与管理等功能，并处理多用户并发连接。
    *   **客户端开发**：使用 **C++** 与 **Qt 6** 进行跨平台客户端开发，实现与服务端的网络通信、界面交互、本地数据缓存，并支持用户各项操作请求。
    *   **核心功能实现**：完成邮箱验证码的生成、有效期控制与校验、账号注册登录，好友添加与管理，消息即时传输，大文件分片传输，聊天记录存储与分页查询，好友在线/离线状态实时更新等；其中验证码邮件的 **SMTP** 投递可由**独立辅助服务**完成，主服务端聚焦鉴权与业务数据一致性。

4.  **系统测试**：对开发的 C/S 架构聊天系统进行整体调试和多场景测试，包括功能测试、性能测试、跨平台兼容性测试和并发测试，确保系统各项功能正常运行，优化网络通信效率和界面交互体验，提高系统的稳定性和易用性。
