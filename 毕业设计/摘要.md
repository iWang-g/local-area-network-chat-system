# 摘要

随着信息技术的发展，局域网已成为校园实验室、小型企业等场景内部信息交互的重要载体，对私密、高效、低依赖公网的即时通信需求持续增长。公网即时通讯工具存在功能冗余、数据出境与隐私风险等问题，而部分简易局域网聊天软件又往往功能单薄、缺乏可靠的身份核验与数据管理能力。针对上述不足，本文设计并实现了一套**基于 C/S 模式的局域网聊天系统**：在局域网内提供账号体系、好友关系与即时通信能力，数据主要在内部网络中传输与落地，兼顾易用性与可控性。

系统采用 **C++** 作为服务端与客户端业务实现的基础语言。**客户端**基于 **Qt 6** 图形框架与网络模块构建，负责界面交互、与服务端的长连接通信及本地体验优化；**服务端**采用标准 C++ 与 **TCP/IP Socket、多线程** 等技术实现连接管理、鉴权、消息路由与数据访问，并可在 **Windows 与 Linux** 环境下编译部署。**数据持久化**采用 **SQLite** 存储用户、好友关系与聊天记录等信息。注册与登录环节引入**邮箱验证码**机制，邮件投递依托 **SMTP**（如 QQ、163 邮箱等）完成；为降低主服务端的协议与依赖复杂度，验证码邮件发送由**独立的轻量级辅助服务**承担，主服务端负责验证码生成、有效期与校验等业务逻辑。

系统实现的主要功能包括：邮箱验证码注册与登录、好友添加与管理、点对点即时文字通信、基于分片策略的文件传输、聊天记录的持久化存储与分页查询，以及好友在线状态的维护与展示。通过 C/S 架构划分客户端与服务端职责，提升了系统的可维护性与并发场景下的处理能力。本课题工作在满足局域网内部即时沟通需求的同时，也为同类跨平台 C/S 通信系统的工程实现提供了可参考的实践路径。

**关键词：** C/S 模式；Qt 6；C++；Socket

# ABSTRACT

With the advancement of information technology, local area networks (LANs) have become a key medium for internal communication in settings such as campus laboratories and small enterprises, where private, efficient instant messaging with reduced reliance on the public Internet is increasingly demanded. Public instant messaging tools often introduce feature bloat and privacy risks associated with off-premises data processing, whereas many lightweight LAN chat utilities remain limited in functionality and lack robust identity verification and data management. To address these gaps, this paper presents a **C/S-based LAN chat system** that provides accounts, buddy relationships, and real-time messaging within a LAN, keeping traffic and stored data largely inside the internal network while remaining easy to use and operable under local control.

The system uses **C++** as the foundation for both server- and client-side business logic. The **client** is built with the **Qt 6** GUI framework and networking modules for UI interaction, long-lived connectivity to the server, and local usability enhancements. The **server** is implemented in standard C++ using **TCP/IP Socket** networking **and multithreading** for connection management, authentication, message routing, and data access, and can be **compiled and deployed on Windows and Linux**. **SQLite** is adopted for persistent storage of users, friend relationships, chat history, and related metadata. **Email verification codes** are used for registration and sign-in; delivery relies on **SMTP** (e.g., QQ Mail, 163 Mail). To keep the primary server simple, outbound mail is handled by a **small standalone auxiliary service**, while the main server owns code generation, expiry, and verification rules.

Major features include email-verified registration and login, friend addition and management, peer-to-peer instant text messaging, chunked file transfer, persistent chat logs with paginated retrieval, and online/offline presence display for buddies. The C/S separation clarifies responsibilities and improves maintainability and concurrent processing under multi-user workloads. Besides meeting practical LAN communication needs, this work offers an engineering reference for cross-platform C/S communication systems of a similar class.

**Key Words:** C/S Mode; Qt 6; C++; Socket
