# 结论

本文围绕“**基于 C/S 模式的局域网聊天系统设计与实现**”这一课题，从需求分析、相关技术、总体设计、协议与数据库设计、系统实现到测试验证，完成了一套可在局域网内部署与使用的即时通信系统。系统以**数据在本地网络内传输与落地**为主要目标，在功能上覆盖**账号与身份核验、好友关系、即时消息、文件传输、历史记录与在线状态**等核心环节，在结构上坚持**客户端与服务端职责分离**，便于维护与扩展。

在技术实现上，**客户端**采用 **C++** 与 **Qt 6**，利用 **Qt Network** 中的 **`QTcpSocket`** 与聊天服务端保持 **TCP** 长连接，并完成图形界面与交互体验方面的实现；**服务端**采用标准 **C++** 与 **TCP/IP Socket、多线程** 等机制完成接入、帧解析、鉴权、消息路由与数据访问，并可在 **Windows、Linux** 环境下分别构建与运行，体现了较好的**可移植性**。**SQLite** 作为嵌入式数据库承担用户、好友关系与消息等结构化数据的持久化；注册登录环节结合**邮箱验证码**，邮件投递通过 **SMTP** 完成，并由**独立的轻量级辅助服务**承担发信，主服务端专注于验证码业务规则与安全策略，降低了主程序在邮件协议上的耦合。

通过功能测试、性能测试与安全性测试可以表明（具体数据以实测为准）：系统在局域网场景下能够稳定完成预定业务流程，接口与存储设计能够与第 4 章所述协议及表结构相对应；性能方面在实验室并发与文件传输条件下可满足一般使用需求；安全性方面在验证码时效、口令哈希存储、参数化 SQL、内网邮件接口鉴权等方面具备基本防护能力。

受时间与个人经验所限，本系统仍存在可改进之处：**传输与存储层面的端到端加密**、**更细粒度的权限与审计**、**群聊与富媒体消息**、**更完善的断点续传与冲突处理**、以及**容器化部署与监控运维**等，均可作为后续工作与研究方向。总体而言，本课题在理论与实践相结合的前提下，完成了从问题分析到系统落地的完整闭环，对同类 **C/S** 架构下的局域网通信软件开发具有一定的参考意义。
