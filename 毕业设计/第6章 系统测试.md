# 第6章 系统测试

本章在模拟**局域网**环境下，从**功能、性能、安全性**等方面对系统进行验证。功能测试按**第 3 章**需求中的四大模块组织用例；性能测试关注并发与传输表现；安全性测试给出**方案**与**结果**对照说明。测试数据中的耗时、占用率等可在定稿时按**实测记录**替换。

---

## 6.1 测试环境与测试工具

### 6.1.1 测试环境

1. **网络拓扑**：多台终端与运行服务端的主机处于**同一局域网**（有线/无线均可），IP 互通，防火墙放行聊天服务端口（如 **8888**，以实现为准）。
2. **硬件环境**（示例）：至少两台客户端终端与一台服务端主机；千兆交换机或等效局域网条件，减少带宽瓶颈对文件传输测试的干扰。（定稿时可填写具体 CPU/内存型号。）
3. **软件环境**（示例）：
   * **服务端**：Windows 11 或 **Ubuntu 22.04**（用于跨平台验证时，可与第 5 章环境一致）。
   * **客户端**：**Windows 11**、**Ubuntu 22.04** 等，安装本系统 **Qt 6** 客户端构建产物。
   * **数据库**：服务端使用的 **SQLite** 数据文件路径已知且可读（仅测试环境，生产应注意权限）。

### 6.1.2 测试工具

| 工具 | 用途说明 |
| :--- | :--- |
| **Navicat Premium 16** | 连接服务端 **SQLite** 库，**实时查看表数据与结构**，核对注册、好友、消息、文件记录等是否与界面操作一致；支持执行简单 **SQL** 辅助排查。（亦可配合导出/筛选功能做抽样校验。） |
| **Wireshark**（如 4.x） | 可选抓取 **TCP** 报文，辅助核对**定长头 + Body** 的帧边界与交互时序（本系统以应用层协议为主，抓包为辅助手段）。 |
| **自研/脚本压测** | 如使用 **Python** 编写多连接客户端或重复登录脚本，模拟**并发接入**（具体脚本与指标见 **6.3**）。 |
| **系统监视器** | 操作系统自带的任务管理器/`top` 等，观察服务端 **CPU、内存、句柄**占用趋势。 |

> **说明**：数据库可视化工具也可选用 **DB Browser for SQLite** 等替代品；本文选用 **Navicat Premium 16** 主要考虑其连接 **SQLite** 便捷、适合在论文中说明“表级联调观察”过程。

1. **并发连接与登录**：使用脚本模拟 **N** 路（如 **30～50**）客户端在短时间内完成建连与登录（具体以实验室条件为准）。观察服务端 **CPU、内存**占用及是否出现**拒绝连接、崩溃**。可将**平均响应时间、最大响应时间**记入表内（示例：在千兆局域网、指定硬件下，登录响应多在 **1.x 秒**量级——**须替换为实测**）。

## 6.4 安全性测试

### 6.4.1 测试方案

安全性测试围绕**身份核验、数据存储、接口与注入、运行环境**展开，方案如下：

| 序号 | 测试方向 | 测试要点 |
| :---: | :--- | :--- |
| S-01 | 验证码机制 | 时效窗口、错误次数限制、用途字段（`purpose`）是否与场景匹配；过期验证码不可完成注册。 |
| S-02 | 口令与存储 | 数据库中口令字段为**不可逆哈希**（加盐），不得以明文落库；客户端传输策略与第 5 章描述一致。 |
| S-03 | 认证与会话 | 未登录或 token 失效时，敏感操作被拒绝；断线后会话清理，防止冒用旧连接。 |
| S-04 | SQL 注入 | 对带输入项的接口（昵称、搜索关键字等）尝试典型注入载荷；服务端使用**参数化查询**时应无异常执行与数据泄露。 |
| S-05 | 辅助服务接口 | 邮件发送 **HTTP** 接口仅监听**内网**，携带**共享密钥**；非法请求被拒绝。 |
| S-06 | 日志与隐私 | 日志中是否误打印口令、验证码明文；生产环境应关闭过度调试输出。 |

### 6.4.2 测试结果

将下列结果在定稿时改为**实测结论**（下表为示例表述模板）。

| 序号 | 测试方向 | 测试结果（示例） |
| :---: | :--- | :--- |
| S-01 | 验证码机制 | 超时后注册被拒绝；连续错误超过阈值后触发限制（若已实现）。 |
| S-02 | 口令与存储 | Navicat 查看 `users` 表，口令字段为哈希密文，无明文。 |
| S-03 | 认证与会话 | 未登录请求被业务层拒绝；断线重连需重新鉴权（与实现一致）。 |
| S-04 | SQL 注入 | 参数化查询下，恶意输入未导致库结构泄露或异常删改。 |
| S-05 | 辅助服务接口 | 无密钥或错误密钥的 HTTP 请求返回失败，不触发发信。 |
| S-06 | 日志与隐私 | 抽查运行日志无完整口令/验证码明文输出（若存在则已整改）。 |

**兼容性说明（补充）**：在 **Windows 与 Linux** 客户端分别登录同一服务端、互发消息与传输文件，界面与编码显示正常、协议互通，可作为验收记录附录，不单独成章。
